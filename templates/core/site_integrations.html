Customer ID (ID do Cliente)

Abra ads.google.com e faça login. O ID aparece no canto superior direito, no formato 123-456-7890.

2

Developer Token (Token de Desenvolvedor)

No Google Ads, vá em Ferramentas e Configurações (ícone de chave inglesa) → Centro de API. Se você não tem uma conta MCC (Manager), crie uma em ads.google.com/manager-accounts. O token aparece na seção API Center.

Nota: Tokens novos começam com acesso de "Teste" (Test Account). Para uso em produção, solicite o upgrade para "Acesso Básico" (Basic Access) no mesmo painel.

3

OAuth Client ID e Client Secret

  1. Acesse o Google Cloud Console → Credenciais
  2. Clique em + Criar Credenciais → ID do cliente OAuth
  3. Tipo de aplicativo: Aplicativo para computador (Desktop app)
  4. Dê um nome (ex: "RankPulse") e clique em Criar
  5. Copie o ID do cliente e o Secret do cliente

Importante: Ative a Google Ads API no projeto. Vá em Biblioteca de APIs → pesquise "Google Ads API" → clique em Ativar.

4

Refresh Token

Este é o passo que autoriza o RankPulse a gerenciar suas campanhas. Use a ferramenta oficial do Google:

  1. Abra o terminal/prompt do seu computador
  2. Execute: pip install google-auth-oauthlib
  3. Execute o comando abaixo substituindo CLIENT_ID e CLIENT_SECRET:
python -c "from google_auth_oauthlib.flow import InstalledAppFlow;
flow = InstalledAppFlow.from_client_config({'installed': {'client_id': 'SEU_CLIENT_ID', 'client_secret': 'SEU_CLIENT_SECRET', 'auth_uri': 'https://accounts.google.com/o/oauth2/auth', 'token_uri': 'https://oauth2.googleapis.com/token'}}, scopes=['https://www.googleapis.com/auth/adwords']);
creds = flow.run_local_server(port=0);
print('Refresh Token:', creds.refresh_token)"

Uma janela do navegador abrirá para você autorizar o acesso. Após autorizar, o Refresh Token será exibido no terminal. Copie e cole aqui.

Testar Conexao

Verifique se as credenciais estao funcionando corretamente antes de criar campanhas.

Erro: invalid_grant

Causa: O Refresh Token expirou, foi revogado, ou o app OAuth esta em modo "Teste".

Solucao:

  1. No Google Cloud Console, verifique se o app esta como "Em producao" (nao "Teste")
  2. Gere um novo Refresh Token usando o botao acima
  3. Se o erro persistir, remova o app das permissoes da conta Google e tente de novo

Erro: DEVELOPER_TOKEN_NOT_APPROVED

Causa: O Developer Token tem acesso de "Teste" e nao pode operar contas reais.

Solucao:

  1. Abra ads.google.com com a conta MCC
  2. Va em Ferramentas → Centro de API
  3. Solicite upgrade para "Acesso Basico" (Basic Access)
  4. Preencha o formulario de aplicacao (leva 1-3 dias uteis para aprovacao)

Erro: API nao encontrada / versao invalida

Causa: A Google Ads API nao esta ativada ou versao da SDK esta desatualizada.

Solucao:

  1. Ative a Google Ads API no Google Cloud Console
  2. Verifique se a versao da SDK google-ads suporta a API v23+

Erro: unauthorized_client ou invalid_client

Causa: Client ID ou Client Secret incorretos, ou o app OAuth foi excluido.

Solucao: Verifique as credenciais no Google Cloud Console → Credenciais. Se necessario, crie um novo cliente OAuth (tipo: Aplicativo para computador).

Dica: Conta MCC (Login Customer ID)

Se sua conta Google Ads e gerenciada por uma MCC (Manager Account), preencha o campo "Login Customer ID" com o ID da conta MCC (formato: 123-456-7890). Sem isso, chamadas a API falham com CUSTOMER_NOT_FOUND.
